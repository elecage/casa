# orbit-propagator (pilot task D)

수학·물리 정답이 정의하는 **숨은 오라클** 과제 (2026-07-23, G2 후 유저
제안으로 추가). 세션은 `template/`의 사본 안에서만 작업한다.

- `template/` — 이체 궤도역학 라이브러리 "orbitlib". `integrate.propagate()`
  미구현 상태로 배포되며(보이는 테스트 2 failed/3 passed), docstring이
  정확도 계약을 명시한다: e ≤ 0.9, 100주기, dt=T/4000에서
  pos ≤ 1e-3·a, |ΔE/E| ≤ 1e-6, |ΔL/L| ≤ 1e-7.
- `hidden_tests/` — 채점 전용 계약 스위트 (러너는 template/만 복사 —
  세션에 절대 노출 금지). **iterate-until-green 포화의 구조적 차단**:
  보이는 테스트에 수렴해도 숨은 스위트에는 최적화할 수 없다.
- 난이도의 원천 (실측, scratchpad/tune_orbit.py):
  | 구현 | e=0.9·100주기 결과 |
  |---|---|
  | Euler (기존 euler_step 재사용) | 전부 실패 (pos 5e+2) — 보이는 테스트도 실패 |
  | 고정 leapfrog | pos 1.26, dE 5.8e-3 → 실패 |
  | **교과서 고정 RK4** | **보이는 테스트 전부 통과, 숨은 계약 실패** (pos 7.3e-2, dE 3.6e-5, dL 7.3e-7) |
  | RK4 + 내부 서브스텝 4 (정답) | 여유 12~140배로 통과 |
  즉 합격에는 "고이심률 근점에서 외부 스텝이 부족하다"는 수치 통찰
  (서브스텝핑/적응 스텝)이 필요하다.
- **변별축 = 자기 검증 행동**: 레포에 analytic(정답해)·energy(보존량)가
  보이게 들어 있다. 계약을 스스로 검증하는 세션(에너지 드리프트를 직접
  측정)은 고이심률 실패를 발견하고, 보이는 테스트만 돌리는 세션은
  통과로 착각한 채 제출한다. 검증 행동은 트랜스크립트에서 관측 가능
  (RQ2의 새 신호 후보).
- 난이도 손잡이: 허용오차 ×배율, e_max, 주기 수, 계약 문구의 힌트 강도.
- `grade.py` — 보이는 pytest + tests/ 변조 검사 + 숨은 스위트
  (PYTHONPATH=workdir/src로 격리 실행, 총 ~10초).

메타 테스트(`tests/test_pilot_orbit.py`) 불변식: 배포 상태 실패 형태,
**교과서 RK4 변형은 보이는 테스트 통과 + 채점 실패** (포화 차단 증명),
정답 전체 통과, 변조 감지.
