# 연구 계획 (Research Plan)

## 배경과 동기

같은 모델, 같은 CLAUDE.md, 같은 프롬프트로도 클로드 코드 세션은 열 때마다
다른 "작업 성격"을 보인다 — 다른 탐색 전략, 다른 수준의 문제 해결 능력.
확률적 샘플링과 서빙 스택 비결정성이 에이전트 루프에서 증폭되고(경로
의존성), 초기의 잘못된 해석이 세션 전체를 오염시키기 때문이다.

선행 연구는 이 분산의 **존재**(pass^k 붕괴, 재현 실패)와 **인프라 원인**
(배치 비불변성 + FP 비결합성)까지는 확립했다(docs/RELATED_WORK.md의
run-to-run 섹션). 그러나 유저가 접근 가능한 수준 — 행동 궤적 — 에서 분산을
**설명**하고, 진행 중인 세션의 성패를 **조기 예측**하는 연구는 실제 코딩
에이전트에 대해서는 없다. 서버측 교정(batch-invariant 커널 등)은 폐쇄 상용
에이전트의 유저에게 불가능하므로, 변동성은 주어진 조건이다. 유저의 실질적
질문은 둘이다: **"이 세션, 계속 갈 만한가 갈아엎어야 하는가?"** 그리고
**"이 세션의 완료 보고를 믿어도 되는가?"** (파일럿 실증: 실패 13세션 중
12세션이 완료를 단언 — docs/PILOT_RESULTS.md §10).

클로드 코드는 전체 트랜스크립트를 로컬 JSONL로 남기고(`~/.claude/projects/`)
훅으로 도구 호출을 가로챌 수 있어, 이 질문에 블랙박스로 답할 수 있는 관측
지점이 존재한다.

## 결과변수 — 세션 평가 3축 (2026-07-24 W10 개정, 유저 승인)

파일럿(W1~W9)의 교훈으로 세션 수준 결과변수를 성공/실패 이분법에서
3축으로 확장한다 (근거: docs/PILOT_RESULTS.md §8~10, STATUS 결정 로그):

1. **실제 달성** — 객관 채점(숨은 오라클 우선; 보이는 테스트는 포화·거짓
   확신을 만든다). 여전히 필수 축.
2. **능력·효율** — 같은 결과를 내는 데 쓴 토큰·턴 수. 같은 과제를 전부
   성공한 세션끼리도 2~4배 차이(파일럿 실측) — 유저가 체감하는 "세션마다
   능력이 다르다"의 정량 대응물. **wall-clock 시간은 서빙 교란(부하·
   대역폭) 때문에 지표로 사용 금지** (2026-07-24 결정).
3. **주장-증거 정합** — 최종 자기 보고(완료 주장)와 트랜스크립트 증거
   (주장 이전 검증 실행, 산출물)의 일치 여부. 허위 완료·무보고 중도
   정지·mock 구현 후 완료 주장을 포괄. 자기 보고 자체는 신호 가치 0
   (성패 무관 균일 확신)이므로 주장이 아니라 **주장-증거 간극**을 잰다.
   casa 코어 지표: `verification_signals`(수정→테스트 사이클, 자기검증
   실행), `claims_completion`, `unverified_completion_claim`.

## 연구 질문

- **RQ1 (변동성의 정량화와 유형화):** 동일 조건(모델·CLAUDE.md·프롬프트·
  레포)에서 반복한 세션들은 얼마나, 어떻게 다른가?
  - 결과 변동: within-task 성공률 분포 (모델은 과제를 "확률 p로 푼다")
  - 행동 변동: 초반 궤적 특징(탐색 폭, 첫 수정까지의 호출 수, 도구 구성비,
    읽은 파일 집합)의 세션 간 분산과 전략 유형(클러스터)의 재현성
  - 준수 변동: 같은 CLAUDE.md 규칙의 준수율이 세션마다 다른가; **규칙을
    어기는 run이 곧 행동적으로 이탈하는 run인가** (준수×일관성 공분산 —
    선행 연구 전무, RELATED_WORK "열린 확인 사항 4")
  - H1b (차등 붕괴): 의무형 규칙("항상 Y 먼저")이 금지형 규칙("X 하지 마라")
    보다 세션 길이에 따라 빨리 붕괴한다
- **RQ2 (조기 예측, 논문의 척추):** 첫 N스텝의 블랙박스 행동 신호만으로
  이 세션의 성패·비용을 언제부터, 얼마나 정확히 예측할 수 있는가?
  (2026-07-24 개정: 예측 대상에 3축 반영 — 실패뿐 아니라 "성공하지만
  고비용" 세션, "허위 완료 위험" 세션 포함)
  - 신호: 첫 수정 전 탐색 폭/커버리지, 도구 에러율, 반복, 규칙 위반,
    **검증 행동(수정→테스트 사이클, 자기검증 실행 — 파일럿 최강 신호,
    PILOT_RESULTS §9)**. 단 파일럿 교훈: 탐색-신호는 과제 유형 의존
    (스위프형에서만 작동, 숨은-오라클형에서는 역방향)
  - 보고: 시점별 AUROC 곡선 (늦은 신호는 예측 가치가 낮다 — 시점이 본질)
    + **calibration (Brier score, ECE)과 임계값에서의 precision/FN율** —
    재시작 결정 규칙은 판별력보다 확률의 정확성에 의존하므로
  - 실용 산출: **재시작 계산법** — 스텝 N에서 실패 확률이 임계 이상이면
    계속보다 새 세션의 기대 비용이 싸다는 결정 규칙. **오탐 비용을 명시적
    포함**: 성공할 세션을 잘못 죽이는 비용(정상 세션 조기 종료율)까지 넣은
    기대 비용으로 계산한다
  - **기준선(이걸 못 이기면 무의미):** ① 항상 계속, ② 고정 토큰 컷오프,
    ③ 단순 휴리스틱(조사 전 수정·동일 명령 반복·테스트 미실행 경고),
    ④ 에이전트 자기평가 (RELATED_WORK의 과신 결과와 연결 — 지표가
    자기평가조차 못 이기면 기여 없음)
  - 핵심 설계: **과제 내(within-task) 반복 샘플링** — 같은 과제를 수십 세션
    돌려 과제 난이도 교란을 제거
- **RQ3 (분기 시점과 경로 의존성):** 성공/실패 세션은 언제부터 구별되는가?
  세션 쌍 간 궤적 유사도가 스텝에 따라 어떻게 벌어지는가? 모의 파이프라인
  에서 보고된 "분기의 60%가 첫 두 호출"(arXiv 2605.28840)이 실제 코딩
  에이전트에서 재현되는가? (파일럿 예비 관찰: 재현 안 됨 — 분리는 초반이
  아니라 중반 이후. PILOT_RESULTS §2)
- **RQ4 (주장-실제 간극; 2026-07-24 신설):** 세션의 완료 주장은 얼마나
  자주, 어떤 조건에서 실제와 어긋나는가? 허위 완료율의 세션 간 분산,
  "검증 없는 완료 주장" 플래그의 정밀도, mock/스텁 구현의 결정론 검출
  가능성. 파일럿 실증: 실패 13세션 중 12건 허위 완료, 1건 무보고 정지;
  자기 보고는 균일 확신(신호 0), 주장 이전 검증 실행 유무가 강신호.
  선행 연구 경계: corrupt success(2603.03116)는 절차-성공 분리, Agentic
  Overconfidence는 자기평가 신뢰도까지 — **최종 보고 주장을 트랜스크립트
  증거와 결정론 대조하는 유저 측 감사는 공백**.

**개념 프레임 (서론용):** "이 세션이 나쁜가"와 "이 과제의 경로가 나쁜가"는
다른 주장이다 — 전자는 세션 수준 잠재 품질(여러 과제에 걸쳐 지속), 후자는
과제별 궤적 속성. 우리 1차 설계(세션당 과제 1개)는 후자를 측정하며 둘을
구분할 수 없음을 명시한다. **부차 실험 arm(본 실험 설계 시 채택 결정):**
일부 세션에 과제 2개를 연속 배치(A→B)해 A 성적이 B를 예측하는지만 확인 —
지속성 가설의 최소 검증. 순서 효과 통제 부담 때문에 1차 설계로는 하지 않는다.

Compaction은 중심 가설이 아니다. compaction 이벤트는 공변량으로 기록만
한다(트랜스크립트에서 무료로 얻어지므로), 유도 실험은 하지 않는다.

## 기여 패키지 (논문 골격)

1. 지속 지시 규칙의 분류 체계: 강제 가능(hook-enforceable) / 측정 가능
   (auditable) / 판단 필요(judgment)
2. 오픈소스 감사 도구 (이 저장소, CASA)
3. 평가 방법론: within-task 반복 샘플링 + 카나리아 규칙 주입 + 검증 가능
   프로브 과제
4. 실증 결과: RQ1의 변동 구조(행동·준수·효율 차원 포함) + RQ2의 조기 예측
   타당성(AUROC 곡선, 재시작 계산법) + RQ3의 분기 동학 + RQ4의 주장-실제
   간극(허위 완료율·검증-주장 정합)

**핵심 원칙: 도구는 기여 목록의 2번이지 주인공이 아니다. 논문의 척추는
실증 발견이다.**

## 발견 강도에 대한 사전 예측 (2026-07 시점의 판단, 파일럿으로 검증할 것)

- RQ1: 분산 존재 자체는 선행 연구로 보아 확실. 기여가 되려면 **구조**가
  나와야 함 — 전략 유형의 재현성, 또는 준수×일관성 공분산.
- RQ2: AUROC 0.6~0.75 예상(중간). 모의 환경의 초반 분기 집중(60%@2스텝)이
  실제 코딩 에이전트로 옮겨온다면 0.75+ 가능. 0.8+면 단독으로 강한 주장.
  약하면 "블랙박스 신호의 한계와 시점 문제"로 정직하게 소화.
- RQ3: 가장 불확실 — 실제 세션의 첫 행동은 탐색적 읽기라 순서가 결과를 덜
  결정할 수 있음. 재현 실패도 그 자체로 모의-실환경 격차 발견.
- 포트폴리오형 연구: RQ1이 바닥을 받치고 RQ2가 상방을 연다.
