# 연구 계획 (Research Plan)

## 배경과 동기

같은 모델, 같은 CLAUDE.md로도 클로드 코드 세션은 세션마다 다른 "작업 성격"을
보인다. 확률적 샘플링이 에이전트 루프에서 증폭되고(경로 의존성), 초기의 잘못된
해석이 세션 전체를 오염시키기 때문이다. 유저가 실제로 겪는 대표적 실패 모드:

1. "레포 전체를 검색해서 답하라"고 했는데 탐색을 충분히 하지 않고 답함
2. CLAUDE.md / memory.md에 명시한 금지·의무 사항을 위반함

이 실패들은 모델 내부 신호 없이 **행동 관찰만으로** 검출 가능하다. 클로드
코드는 전체 트랜스크립트를 로컬 JSONL로 남기고(`~/.claude/projects/`),
훅(PreToolUse/PostToolUse/Stop)으로 도구 호출을 가로챌 수 있다.

## 연구 질문

- **RQ1 (지속 지시 준수):** 세션 내내 유지되어야 하는 설정 파일(CLAUDE.md)의
  지시를 에이전트가 실제로 얼마나 지키는가? 준수율은 세션 길이, 컨텍스트
  압박, 압축(compaction) 이벤트에 따라 어떻게 변하는가?
  - H1a (압축 절벽): 위반율은 compaction 이벤트 직후 계단식으로 상승한다.
  - H1b (차등 붕괴): 의무형 규칙("항상 Y 먼저")이 금지형 규칙("X 하지 마라")
    보다 빨리 붕괴한다.
- **RQ2 (블랙박스 실패 예측):** 유저가 접근 가능한 행동 신호만으로 세션
  실패를 얼마나 조기에 예측할 수 있는가?
  - 신호: 첫 수정 전 탐색 폭/커버리지, 반복 루프, 도구 에러율, 백트래킹.
  - 핵심 설계: **과제 내(within-task) 반복 샘플링** — 같은 과제를 수십 세션
    돌려 성공/실패 세션의 초반 궤적을 비교. 과제 난이도 교란을 제거한다.
  - 주의: 늦은 신호(반복 루프)는 예측 가치가 낮다. 초반 N스텝 신호의
    AUROC를 시점별로 보고할 것.

## 기여 패키지 (논문 골격)

1. 지속 지시 규칙의 분류 체계: 강제 가능(hook-enforceable) / 측정 가능
   (auditable) / 판단 필요(judgment) — 어떤 규칙이 어디 속하는지의 태소노미
2. 오픈소스 감사 도구 (이 저장소, CASA)
3. 평가 방법론: 카나리아 규칙 주입 + 검증 가능 프로브 과제
4. 실증 결과: RQ1의 준수율 구조 + RQ2의 예측 타당성(AUROC)

**핵심 원칙: 도구는 기여 목록의 2번이지 주인공이 아니다. 논문의 척추는
실증 발견이다.**

## 발견 강도에 대한 사전 예측 (2026-07 시점의 판단, 파일럿으로 검증할 것)

- RQ1: 구조적 발견(압축 절벽 또는 차등 붕괴) 확률 높음. 단순 "길수록 위반↑"만
  나오면 통념 확인 수준 — 구조가 나와야 강한 논문.
- RQ2: 초반 신호 AUROC 0.6~0.75 예상(중간). 0.8+ 이면 단독으로 강한 주장.
  약하게 나오면 "블랙박스 신호의 한계와 시점 문제"로 정직하게 소화.
- 포트폴리오형 연구: RQ1이 바닥을 받치고 RQ2가 상방을 연다.

## 리스크와 방어

| 리스크 | 방어 |
|---|---|
| 일반화 ("클로드 코드 하나?") | 훅·로그 구조가 있는 다른 에이전트 CLI(예: Gemini CLI) 1~2개 교차 검증 |
| 재현성 (폐쇄 모델의 업데이트) | 모델 버전 기록, 조건당 수십~수백 세션, 통계 처리. "버전이 바뀌어도 감사 방법론은 유효"로 프레이밍 |
| 성공/실패 라벨 신뢰성 | LLM 심판 단독 사용 금지 (정밀도 ~70%). 테스트 통과 등 객관 기준이 있는 과제만 사용 |
| 과제 난이도 교란 (RQ2) | within-task 반복 샘플링 설계 |
