# 파일럿 설계 (Pilot Design)

목적: 본 실험에 투자하기 전에 **발견의 강도**를 데이터로 추정한다.
확인할 세 가지:

1. within-task 세션 분산의 크기 — 성공률이 분산 관찰에 유효한 구간(40~80%)
   에 있고, 행동 지표가 세션 간에 실제로 흩어지는가 (RQ1 예비)
2. 첫 수정 이전 행동 지표만으로 성공/실패가 분리되는가 (RQ2 예비)
3. 성공/실패 세션의 궤적이 몇 스텝째부터 구별되는가 (RQ3 예비)

## 규모

- 카나리아 규칙 8개 (`rules/canary_rules.yaml`)
- 과제 3종 × 세션 15~20개 = 45~60 세션 — 과제별 구체 설계는
  `docs/PILOT_TASKS.md` (buggy-pipeline / plugin-add / rename-sweep)
- 과제 조건: 객관적 성공 판정 가능(테스트 통과 등), 세션당 20분 이내,
  중간 난이도(성공률 40~80% 구간이 분산 관찰에 최적)
- 본 수집 전 과제당 2~3개 보정 세션으로 난이도 확인, 40~80% 밖이면 버그
  개수/미묘함으로 조정

## 카나리아 규칙 설계 원칙

- 모든 규칙은 트랜스크립트에서 **기계 판정 가능**해야 한다
- 금지형(prohibition)과 의무형(obligation)을 짝지어 배치 → H1b 차등 붕괴 검정
- 과제 수행에 자연스럽게 걸리는 규칙이어야 한다 (한 번도 트리거되지 않는
  규칙은 데이터가 없다). 각 과제는 규칙이 걸릴 상황을 의도적으로 포함할 것
- 파일럿에서는 훅 차단을 **끄고 기록만** 한다 (위반을 관찰해야 하므로,
  PreToolUse 훅은 log-only 모드)

## 절차

1. 대상 레포에 카나리아 CLAUDE.md 배치 (`scripts/` 참고)
2. 동일 프롬프트로 세션 반복 실행 (모델 버전, 클로드 코드 버전 기록)
3. 세션 종료마다 `casa audit <transcript.jsonl> --rules rules/canary_rules.yaml`
4. 세션별 산출: 위반 목록(규칙 ID, 스텝 인덱스), 행동 지표 시계열,
   과제 성공 여부(외부 채점 스크립트)

## 중간 점검 게이트

판정 기준(아래)은 파일럿 종료 시점의 최종 관문이고, 그 전에 방향 점검
게이트 G1(수직 슬라이스: 헤드리스 실행 파이프라인 생존성)과 G2(보정:
분산 존재 확인)를 둔다. 게이트 정의·고정 질문 5개·기록은 `STATUS.md`의
"방향 점검 게이트" 절. 게이트 통과 없이 다음 단계 비용을 쓰지 않는다.

## 판정 기준 (사전 등록)

- **RQ1 유망**: 과제별 성공률이 40~80% 구간 & 초반 행동 지표(탐색 폭,
  커버리지)의 세션 간 변동계수가 유의미 (분산이 바닥이면 예측할 것도 없다)
- **준수×일관성**: 규칙 위반 수와 행동 지표 이상치 여부의 세션 수준 상관을
  예비 보고 (방향만 확인, 검정은 본 실험)
- **H1b 지지**: 의무형 위반율 > 금지형 위반율, 격차가 세션 길이에 따라 확대
- **RQ2 유망**: 첫 수정 이전 지표만으로 성공/실패 AUROC ≥ 0.7 (within-task)
- **RQ3 예비**: 성공/실패 군의 궤적 특징이 세션 중반 이전에 분리되는지 관찰
  (모의 환경의 "60%가 첫 2스텝" 재현 여부 기술)
- 모두 약하면: 본 실험 보류, 결과를 워크숍 페이퍼/블로그로 정리 후 재설계

## 측정 지표 (casa가 자동 산출)

세션 요약값:

- exploration_before_first_edit: 첫 Edit/Write 전 탐색 호출 수
- coverage: 답변 전 읽은 파일 수 / 관련 파일 수 (과제별 관련 파일 목록 사전 정의)
- repetition: 동일 도구 호출의 최대 반복 횟수
- tool_error_rate: is_error 결과 비율
- rule_violations: 규칙 ID별 위반 (금지형: 발생 즉시 / 의무형: 트리거 시점에
  선행 조건 미충족)
- compaction_count: 공변량으로 기록만 (중심 가설 아님)

궤적 수준 (RQ2 시점별 AUROC·RQ3 분기 분석용 — 지표 확장 필요):

- 스텝별 누적 신호: k스텝 시점의 탐색 폭/커버리지/에러율/위반 수 (k를 훑으며
  AUROC@k 곡선 산출)
- 세션 간 궤적 유사도: 도구 호출 시퀀스의 정규화 prefix 편집 거리
  (선행 연구의 TSS/prefix edit distance와 비교 가능하게)
